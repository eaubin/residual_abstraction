# Agent Workflow Automation

This repository's experiment process uses two roles:

- **worker**: writes or revises the experiment artifact, implementation, output,
  conclusion, and commits;
- **reviewer**: performs the repository experiment review and either approves or
  requests changes.

`EXPERIMENT_REVIEW_PROTOCOL.md` remains canonical for review standards. The
scripts here only reduce copy/paste and preserve transcripts.

## Files

| path | purpose |
|---|---|
| `scripts/agent_harness.py` | CLI harnesses for Codex and Claude conversations |
| `scripts/experiment_agent_loop.py` | worker/reviewer preregistration and result loops |
| `.agent_runs/` | ignored transcript directory generated by loop runs |

The old `todo/` demo files were adapted into these repo-specific scripts and
removed after adaptation.

## Basic Usage

Dry-run the prompts without invoking any agent:

```bash
uv run python scripts/experiment_agent_loop.py prereg \
  --slug exp30-oblique-preflight \
  --task "Prepare I0 from INTERVENTION_CLASS_BENCHMARK.md." \
  --dry-run
```

Run a preregistration loop with Codex as worker and reviewer:

```bash
uv run python scripts/experiment_agent_loop.py prereg \
  --slug exp30-oblique-preflight \
  --worker codex \
  --reviewer codex \
  --task "Prepare I0 from INTERVENTION_CLASS_BENCHMARK.md."
```

Run with Claude as reviewer:

```bash
uv run python scripts/experiment_agent_loop.py prereg \
  --slug exp30-oblique-preflight \
  --worker codex \
  --reviewer claude \
  --task-file prompts/exp30-task.md
```

After preregistration is approved and the experiment is ready to run, use the
result loop:

```bash
uv run python scripts/experiment_agent_loop.py result \
  --slug exp30-oblique-preflight \
  --worker codex \
  --reviewer codex \
  --task "Run the approved exp30 preregistration and write the conclusion."
```

## What The Script Automates

For `prereg` mode, the worker is prompted to:

- orient in the repo;
- create both the experiment writeup and runnable script;
- include guards, self-checks, output tables, verdict predicates, and halt
  conditions;
- commit the preregistration state;
- stop before the first claim-producing run.

The reviewer is prompted to:

- use `EXPERIMENT_REVIEW_PROTOCOL.md`;
- check phase fit, construct validity, verdict partitioning, scope indexing,
  anti-vacuity, ground-truth discipline, LLM-work creep, and maintainability;
- return exactly `REVIEW_DECISION: APPROVED` or
  `REVIEW_DECISION: CHANGES_REQUESTED`.

For `result` mode, the worker is prompted to run the approved experiment, write
outputs/conclusions, propagate resolved claims across docs, and commit. The
reviewer then applies the result-review half of the protocol.

## What The Script Does Not Automate

- It does not decide the scientific question.
- It does not make reviewer findings correct.
- It does not guarantee Claude reviewer turns are read-only. Codex reviewer
  turns use a read-only sandbox; Claude reviewer turns rely on the reviewer
  prompt and local CLI permissions.
- It does not replace the pre-registration pause. Approval is still an explicit
  reviewer marker.
- It does not commit transcripts; `.agent_runs/` is ignored by default.

## Prompt Orientation

Each role receives a standing orientation that tells it to read or inspect:

- `AGENTS.md`;
- `EXPERIMENT_REVIEW_PROTOCOL.md`;
- `INTERVENTION_CLASS_BENCHMARK.md`;
- `HANDOFF.md`;
- `EXPERIMENTS.md`, especially exps 10-17 and 23-29;
- `experiments/29-predicate-targeting.md` and `out/exp29_pstack-L4.txt`;
- exps 10-17 when read/write intervention history is load-bearing;
- `FORMALISM.md` section 6.1 when verdict partitions or tolerance branches
  appear.

This captures the repeated manual setup: orient in the repo, pay attention to
experiments 10-17, preserve experiment-local/repo-local/widely-used terminology,
and use the review protocol rather than generic code review.

## Operational Notes

- Use `--task-file` for anything longer than a sentence or two.
- Real worker loops require a clean working tree by default; pass
  `--allow-dirty` only when the current uncommitted state is intentional.
- Use `--max-rounds` to bound unattended loops.
- Use `--codex-danger` only inside an external sandbox. The default Codex worker
  uses workspace-write sandboxing.
- Inspect `.agent_runs/<timestamp>-<mode>-<slug>/` when an agent fails to follow
  markers or when you need to resume manually.
- If a loop stops without approval, use the last reviewer transcript as the next
  worker task or run another loop with a narrower task.
